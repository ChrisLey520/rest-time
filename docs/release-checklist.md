# 栖时发布清单

发布前逐项确认。当前目标：AppGallery 首发 0.1.0。

## 1. 版本与签名

- [ ] `AppScope/app.json5` 确认 `versionCode` / `versionName`（当前 1000000 / 0.1.0）。
- [ ] `bundleName` 为 `com.qishi.focus`，与 AGC 项目一致。
- [ ] 在 DevEco Studio `File > Project Structure > Signing Configs` 配置发布证书（.p12 + .cer + .p7b）。
- [ ] 用 release profile 构建签名 HAP：`node scripts/check-deveco-env.mjs --assemble`。
- [ ] 确认签名 HAP 能在真机安装启动（`hdc install`）。

## 2. 隐私与合规

- [ ] 隐私政策文本（docs/privacy-policy.md）发布到可公开访问的 URL，AGC 提交时填写。
- [ ] AGC 隐私标签按政策填写：不收集数据、无第三方 SDK。
- [ ] 首次启动的通知权限弹窗文案确认（当前由系统提供）。
- [ ] 代理提醒权限（PUBLISH_AGENT_REMINDER）在 module.json5 已声明，AGC 权限说明填写"专注到点提醒"。

## 3. 应用市场素材

- [ ] 应用名称：栖时。
- [ ] 一句话简介：帮你在 30 秒内进入下一段专注，并把时间回收到进度和复盘里。
- [ ] 描述文案要点：下一段建议 / 保底启动 / 退出救援 / 前台专注凭证 / 时光币 / 月度复盘。
- [ ] 截图（真机分辨率）：首页启动台、专注中小动作、完成后闭环、记录页分层、月度报告。
- [ ] 图标 256px 已就位（AppScope + entry 资源）。

## 4. 功能回归（真机）

- [ ] 首页 30 秒内可开始第一段专注（无任务 → 写下一件事并开始）。
- [ ] 保底启动：制造 2 次放弃后，首页出现 5 分钟保底启动卡片。
- [ ] 中途退出救援：点击终止 → 选择原因 → 20 秒冷静期 → 确认终止记为失败。
- [ ] 离开即失败：专注中切后台/锁屏/杀进程，回来后本轮失败且不可恢复结算。
- [ ] 服务卡片：添加 2*2 卡片，专注中状态同步，失败后不再显示"进行中"。
- [ ] 系统提醒：专注到点后收到"回来确认这一段"，点击进入 App。
- [ ] 时光币：5 分钟以上完整专注即发放（25 分钟默认专注约 25 币），中断/离开不发放。
- [ ] 记录页分层：清数据后只见引导；完成 1 段后见今日/本周；活跃 3 天后见月报。
- [ ] CSV 导出：记录页导出按钮拉起文件选择器，保存后文件可用表格软件打开（无乱码）。
- [ ] 长图导出：月报卡片"保存长图"按钮生成 PNG 并保存到相册，图片内容完整。
- [ ] 客户账单：本月/上月切换正确，发票备注显示在卡片和 CSV。
- [ ] 异常日志：设置页开发调试区能看到提醒/卡片失败记录（如有）。
- [ ] 清除本地数据后应用回到全新状态且不崩溃。

## 5. 已知边界（首发可接受）

- 时光币仅累计展示，兑换体系未上线。
- 长图导出为月报卡片当前可见区域截图，超长内容以卡片实际渲染为准。
- 无云同步、无账号——数据仅本地，卸载即丢，描述文案中需提示用户导出备份。
- Pro 能力全部本地预览，未接支付。
