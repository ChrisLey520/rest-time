# 时光币用途方案

时光币当前只做累计展示。本文基于竞品调研和行为研究，给出用途设计方案和推进顺序。

## 一、竞品怎么用虚拟货币

### Forest（金币）

- 解锁 90+ 树种和白噪音：第一种 500 币、每种递增 100 币、1000 币封顶。
- 兑换真树：与 Trees for the Future 合作，用户累计种下 190 万+ 棵真树——把虚拟奖励接到真实世界意义。
- 删除枯树（撤销失败痕迹）：价格昂贵，本质是"付费消除愧疚"。
- 教训：金币获取过慢导致大量"改系统时间刷币"攻略流传——稀缺性带来激励强度，也带来作弊动机。

### Duolingo（宝石）

- 核心用途是买"道具"：Streak Freeze（连胜冻结保险）、XP Boost、Timer Boost。
- 里程碑宝箱发放宝石/冻结/专属收藏品，Streak Society 从单一 365 天门槛改为 7 天起的多层级。
- 效果（2025 数据）：streak 活跃用户回访概率 3 倍，DAU/MAU 约 37%，月留存 55%。
- 关键设计：货币的最大用途是**保护用户的连续性资产**（loss aversion + 保险），而不是装饰。

### Habitica（金币）

- 买游戏内装备 + **用户自定义现实奖励**（"买杯咖啡""看场电影"），定价建议按"几天的工作量"折算。
- 自定义奖励同时是自我限制工具：想做的娱乐要用金币买，变相配额化。
- 教训：金币产出高度可自定义，导致"赚得多 ≠ 更高效"，价格体系需要用户自己校准，门槛偏高。

## 二、设计原则（结合栖时气质）

1. **不装饰化**：栖时没有虚拟形象/森林，不走皮肤路线的重资产美术。
2. **用途必须服务主线**：花币的行为要让用户更容易开始、坚持或回收，而不是制造第二个游戏。
3. **保护而非惩罚**：Duolingo 验证过的最强用途是"给失败上保险"——与栖时"失败有救援"的定位完全同构。
4. **防通胀**：已有单次 120/单日 240 上限；所有兑换品定价按"天"折算（参考 Habitica 的定价启发式）。
5. **不碰钱**：时光币永远不能购买 Pro 或参与支付，避免虚拟货币合规风险（应用市场对可购买虚拟币有额外审核要求；时光币只能赚不能充值，保持"成就资产"属性）。

## 三、用途方案（按优先级）

### P0：节奏保护券（对齐 Duolingo Streak Freeze）✅ 已实现

- 用途：额外的休息日。当前弹性节奏每 7 天内置 1 个免费休息日；用币可兑换 1 张"节奏保护券"，缺席时自动消耗，节奏不断。
- 定价：120 币/张（约一次长专注或一天的一半上限），持有上限 2 张——保证"缺席仍有成本"，不至于让节奏统计失真。
- 实现说明：`streakGuardCore`（余额=累计-已花、兑换校验、天状态机 focus/blank/covered）+ `StreakGuardService`（消耗时把被覆盖日期写入 `coveredDayStarts` 持久化，保证重算不重复扣券）+ 记录页兑换入口。

### P1：专注氛围解锁（轻量版 Forest 树种）

- 用途：解锁专注中的主题色/背景纹理/完成动效（纯色彩与文案变化，无重美术）。
- 定价建议：首个 300 币，递增 100，600 封顶（比 Forest 便宜，栖时的币产出更慢）。
- 价值：给长期用户"攒币有盼头"，且不影响功能公平。

### P2：自定义现实奖励（Habitica 模式，轻量做）

- 用途：用户自己写奖励（"看一集剧 = 60 币"），完成兑换时打卡消费。
- 价值：把币接到真实生活，特别适合备考/自由职业用户自我管理。
- 风险：需要引导定价，做成模板（默认给 3 个建议价格），避免 Habitica 的校准负担。

### 暂不做

- 真树公益兑换：有品牌价值但需要商务合作与资金流，留到有收入后。
- 排行榜/社交比拼：与「安静」气质冲突，且研究显示纯时长比拼会扭曲行为。
- 币购 Pro/抵扣付费：合规风险 + 破坏订阅定价，永久不做。

## 四、防刷与合规备忘

- 改系统时间刷币（Forest 已验证的漏洞）：发放时校验 `endedAt` 单调性，回拨时间的记录不发币；上限逻辑已天然限制收益。
- 币余额进 StorageService 本地存储，导出明细已含 `timeCoinsEarned`，兑换记录同样要可导出可审计。
- 上架描述中避免"货币""充值"字样，统一叫"时光币成就"。

## 五、推进顺序建议

1. 本周期：实现 P0 节奏保护券（core 纯函数 + 测试 + 首页/记录页入口）。
2. 下周期：P1 氛围解锁（先做 3 个主题色验证兑换漏斗）。
3. 观察指标：兑换率、兑换后 7 日留存对比未兑换用户、保护券实际消耗率。
4. 数据支持后再决定 P2 和公益兑换。
