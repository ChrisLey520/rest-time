# 栖时商业化路线

本文档用于把栖时从本地 MVP 推进到可商业化产品。当前方向不是把计时器做得更复杂，也不是堆报表和账单，而是把「下一段专注」变成一个可持续付费的闭环。

产品核心判断见 [product-core.md](product-core.md)。所有商业化能力必须服从这条主线：用户打开栖时后，能快速知道下一段做什么；当用户想退出时，产品要先救援；完成后的时间要回收到进度、复盘和必要的交付材料里。

## 产品定位

栖时面向需要自主管理时间的知识工作者和创作者，核心价值是帮用户把混乱的一天切成下一段可开始、可完成、可复盘的专注。

### 一句话

一个帮你在 30 秒内进入下一段专注，并把时间回收到进度和复盘里的工具。

### 目标用户

- 自由职业者、学生、创作者：需要自主管理长时段工作，但经常卡在“现在先做什么”。
- 产品、研发、设计等知识工作者：需要在碎片会议之间保护深度工作时间，并把任务切成可完成的小段。
- 轻度自律用户：不想维护复杂项目管理软件，只想降低开始阻力，完成今天第一段。

## 商业化假设

免费版解决「今天开始并完成第一段专注」的问题，付费版解决「长期稳定、可复盘、可交付」的问题。

| 层级 | 用户问题 | 免费能力 | 付费能力 |
| --- | --- | --- | --- |
| 计划 | 我下一段该做什么 | 今日计划、任务优先级 | 周计划、智能结转、计划模板 |
| 执行 | 我能不能现在开始并坚持 | 专注计时、歇息、提醒 | 深度工作模式、跨端提醒 |
| 记录 | 我今天做了多久 | 今日统计、历史记录 | 月度/季度报告、图片导出 |
| 复盘 | 我的效率什么时候更好 | 最近 30 天高效时段 | 多维标签、项目对比、趋势解释 |
| 变现 | 我这个月能结多少钱 | 项目归属、基础投入统计 | 客户账单、计费规则、对账导出 |
| 连续性 | 我能不能坚持下去 | 连续天数、目标进度 | 多设备同步、备份、跨端提醒 |

## 版本分层

### Free

- 专注计时、歇息、系统提醒。
- 任务创建、预计专注次数、完成/归档。
- 首页下一段建议、自由专注和从任务开始专注。
- 今日目标、本周概览、最近 30 天基础洞察。
- 2*2 服务卡片。

### Pro

- 月度复盘报告：完成率、连续天数、最佳时段、任务投入分布。
- 数据导出：CSV、图片长图、复盘摘要。
- 客户账单：按项目汇总可结算金额、计费时长、费率和取整规则。
- 周计划：本周容量、每日建议段数、未完成任务结转。
- 深度标签：场景、能量等级、任务类型；项目/领域已进入基础任务模型。
- 多设备同步和备份。

### Team / Studio

- 团队共享专注时段。
- 个人数据私密、团队只看聚合趋势。
- 项目维度投入统计。
- 组织节奏建议，例如会议密度与深度工作窗口。

## 90 天路线

### 当前进度

- 已完成“下一段专注”基础闭环：任务/自由专注开始、完成记录、任务进度、歇息、今日目标和历史记录。
- 已完成开局防崩保底启动：根据最近放弃、离场和完成率自动推荐 5/10/15 分钟启动段，降低用户一开始就被完整 25 分钟劝退的概率，启动风险和原因会进入记录与导出。
- 已完成专注内微行动：开始专注时生成 3 分钟小动作，用户可完成、调小或触发卡住救援，降低只靠毅力硬撑的失败率。
- 已完成中途退出救援：终止本次前先询问退出原因并进入 20 秒冷静期，提供回去继续或确认终止；确认终止即失败、无时光币。
- 已完成退出交接：确认放弃前会把当前小动作转换成下一次可恢复入口，写入记录和导出，减少用户空手退出后的复启成本。
- 已完成失败恢复入口：最近一次放弃后，专注页提供 1/3/5 分钟恢复阶梯，完成记录会保留重启来源和是否计入正式专注，帮助用户从低谷回到行动但不污染任务/账单数据。
- 已完成后台防刷、硬专注守门和前台专注凭证：专注中离开 App、锁屏、心跳断档或后台被终止会立即失败，不能暂停恢复、不能结算、不能获得时光币；守门模式、白名单完整性、违规原因、离开次数、离开时长、恢复阶梯和时光币写入记录与导出明细。
- 已完成时光币奖励雏形：30 分钟以上完整硬专注才可获得时光币，随时长递增，并设置单次与单日上限；失败、终止、离开和恢复阶梯不发放。
- 已完成完成后闭环：完成一段后可以开始歇息、继续下一段、进入记录复盘或结束今天。
- 已完成今日计划、执行看板、复盘洞察和 Pro 预览。
- 已完成规划核心，能根据任务优先级、计划档期、剩余专注次数、目标缺口和高效时段生成下一步建议。
- 已完成任务优先级与计划档期字段，支持创建、编辑和列表展示。
- 已完成任务项目/领域字段，专注记录会保留项目归属，用于长期投入分析。
- 已完成月度报告本地原型，展示本月总计、完成次数、活跃天数、完成率、平均完成时长、项目投入分布和导出入口。
- 已完成月度报告可导出数据结构，包含摘要、洞察、建议、指标矩阵和项目分布，后续 CSV/图片导出可直接复用。
- 已完成客户账单 Pro 原型，可根据小时费率、最小计费分钟和取整粒度，将完成专注记录折算为本月可结算金额，并按项目汇总贡献排行。
- 已完成 CSV 导出核心，可生成月度报告指标、项目投入分布、客户账单和带启动护栏/退出交接/离开审计/恢复阶梯/时光币/小动作次数/重启来源/是否计入专注的专注记录明细四份 CSV 数据。
- 已完成一键计划模板，帮助用户从空白状态快速生成深度工作日、学习冲刺、维护收尾和轻量重启任务组。
- 已完成今日队列筛选和快速调度，用户可以按今天、明天、未计划、高优先聚焦任务，并一键调整计划档期。
- 已完成记录页阶段分层：无记录只显示第一段引导；30 天内活跃不足 3 天只显示今日/本周和解锁提示；活跃满 3 天解锁月报；再完成 3 次带项目归属的专注才展示客户账单。
- 已完成首页推荐深化：下一段建议附带推荐理由（优先级/档期/只差一段）和本段收益（进度推进/收尾提示），首页开始按钮与规划核心排序一致。
- 已完成 CSV 导出落地：接入系统文件选择器，月报指标、项目投入、客户账单、专注明细四份 CSV 可真实保存，带 BOM 防乱码。
- 已完成客户账单深化：本月/上月周期切换、项目不可结算标记、发票备注（设置页可编辑，账单卡和 CSV 同步展示）。
- 已完成月度报告深化：新增最佳连续天数和环比上月投入趋势指标。
- 已完成任务项目快选：从历史任务和记录提取常用项目，新建/编辑表单一键填入。
- 已完成 Pro 入口整理：ProService 统一能力清单和订阅状态占位，设置页动态渲染。
- 已完成异常日志：提醒失败、通知失败、卡片刷新失败、导出失败和前台恢复失败会写入本地环形日志（上限 50 条），设置页开发调试区可查看和清空。
- 已完成隐私政策（docs/privacy-policy.md）和发布清单（docs/release-checklist.md）。
- 已完成月度报告长图导出：componentSnapshot 截取月报卡片打包 PNG，经系统图片选择器保存，失败路径全部写入异常日志。
- 已完成项目颜色标识：项目名哈希稳定映射到 8 色柔和色板，任务列表、项目快选、记录列表、月报和账单项目行统一显示项目色点，进度条同色。

### 第 1 阶段：商品感和留存闭环

- 强化首页“下一段专注启动台”：推荐任务、推荐理由、开始按钮和空状态一屏完成。
- 完成今日计划、执行看板、复盘洞察、Pro 预览。
- 新增规划核心：今日剩余、明日结转、本周容量。
- 优化空状态，让新用户知道第一步应该创建任务还是自由专注，并能立即开始。
- 真机验证服务卡片、提醒、离开即失败和后台终止记录。

### 第 2 阶段：可付费能力原型

- 先验证 7 天内稳定记录，再突出 Pro 能力。
- 月度复盘页面：先做本地计算和静态入口，不急接支付。
- 数据导出：优先 CSV，再做图片报告。
- 客户账单：先完成本地计费规则、项目聚合和导出，再做客户筛选/发票备注。
- 任务标签：先做本地字段和筛选，再扩展统计。
- 计划模板：工作日、学习日、轻量维护日。

### 第 3 阶段：发布准备

- 签名、隐私政策、应用市场素材。
- 新手引导和权限说明。
- 异常恢复：提醒失败、存储失败、卡片刷新失败的用户反馈。
- 埋点方案：仅记录必要聚合事件，不上传任务标题等敏感文本。

## 产品指标

| 阶段 | 指标 | 目标 |
| --- | --- | --- |
| 开始 | 新用户首次开始专注耗时 | 30 秒内 |
| 激活 | 首日完成 1 次专注 | 50%+ |
| 留存 | 次日再次打开 | 30%+ |
| 习惯 | 7 天内完成 3 天专注 | 20%+ |
| 归属 | 有任务或项目归属的专注占比 | 40%+ |
| 付费 | 查看 Pro 入口后触发意向 | 5%+ |
| 价值 | 月度报告打开率 | 35%+ |

## 当前工程优先级

1. 真机验证：按 docs/release-checklist.md 第 4 节回归——服务卡片刷新、提醒权限、离开即失败、后台终止记录、CSV/长图导出和 unsigned/sign HAP 安装路径。
2. 发布收尾：签名配置、隐私政策 URL、应用市场截图（清单第 1-3 节）。
3. 任务规划深化：模板自定义。
4. 商业接入：订阅支付打通，替换 ProService 本地占位。

## 风险

- 功能堆叠会稀释「安静」气质，规划模块必须轻，不要变成项目管理系统。
- 首页如果不能让用户立刻开始下一段，其他商业功能都会失去根基。
- 数据建议不能显得武断，文案要保持建议性质。
- 付费入口不能阻挡基础计时，否则会伤害信任。
- 同步和账号会显著增加复杂度，应在本地产品价值跑顺后再做。
