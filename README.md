# 栖时

栖时是一个 HarmonyOS 原生专注计时 App 原型。它的核心不是把计时器做复杂，而是帮助用户在不知道下一段该做什么、容易拖延和失控时，快速进入一段明确、有边界、可完成、可复盘的专注。

产品核心判断见 [docs/product-core.md](docs/product-core.md)：用户打开栖时后，应能在 30 秒内知道下一段做什么并开始专注；完成后，这段时间要自动回收到任务进度、记录、复盘和必要的项目归属里。

当前仓库已经实现可交互的本地 MVP：专注计时、任务、记录、统计、设置、系统提醒和服务卡片骨架。

## 当前能力

- 应用名：栖时
- 包名：`com.qishi.focus`
- 底部页面：专注 / 任务 / 记录 / 设置
- 专注页支持开始硬专注、终止本次、完成后歇息和失败后恢复阶梯；专注开始后不能暂停，只能终止
- 设置页可开启「开发调试」后显示「快速验证 10 秒」，用于真机调试完成链路
- 专注时长读取设置，默认 25 分钟
- 完成专注后进入待歇息状态，可开始短歇息或跳过
- 支持按配置在固定轮次后进入长歇息
- 支持配置每日专注目标，专注页展示今日目标进度和剩余目标时长
- 专注页新增今日计划工作台，展示下一步建议、待办投入、连续专注和高效时段
- 专注页新增规划快照，展示今日建议专注次数、明日结转和本周剩余投入
- 专注页在无任务时支持「写下一件事并开始」，自动创建 1 次专注的今日任务并立即进入计时
- 专注页新增「保底启动」，会根据最近放弃、离场和完成率自动推荐 5/10/15 分钟启动段；用户仍可手动切回完整专注，启动风险和原因会进入记录与导出
- 专注进行中会显示 3 分钟「下一步小动作」，支持完成小动作、换成更小动作和卡住救援，让低毅力用户不用硬撑完整 25 分钟
- 专注中点击「终止本次」会先询问退出原因并进入 20 秒冷静期，可回去继续或确认终止；确认终止会失败、无时光币，并留下退出交接，降低随手空手退出
- 最近一次专注如果已结束/放弃，专注页会提供「1/3/5 分钟恢复阶梯」，优先回到原任务，把失败后的下一次打开变成低门槛恢复；恢复记录可见但不计入正式任务进度、今日目标或客户账单
- 专注中切走、锁屏、心跳断档或后台被终止会立即让本轮失败且无时光币；硬专注守门强制开启，只允许栖时前台白名单，并在专注页、服务卡片、记录和导出里标明违规原因
- 完成 30 分钟以上的完整硬专注可获得时光币；时长越长奖励越多，但有单次上限和单日上限，终止、离开和恢复阶梯均不发放时光币
- 完成专注后展示完成反馈和当前专注轮次
- 完成专注后提供「开始歇息 / 继续下一段 / 记录复盘 / 结束今天」闭环选择，避免用户完成后不知道下一步
- 任务页支持创建任务、编辑任务、设置预计专注次数、开始任务专注、标记完成、归档、恢复和删除
- 任务支持项目/领域、优先级和计划档期，任务页可创建、编辑并展示项目归属、「高优先 / 普通 / 低优先」和「今天 / 明天 / 以后 / 未计划」
- 任务页新增一键计划模板，可快速生成深度工作日、学习冲刺、维护收尾、轻量重启等任务组
- 任务页新增执行看板，汇总待办/完成/归档数量、整体进度和优先推进任务
- 任务页复用规划快照，展示今日建议、明日结转和本周剩余专注次数
- 任务页新增今日队列筛选，可按全部、今天、明天、未计划、高优先快速聚焦任务
- 活跃任务支持快速调度，可一键切换今天/明天/未计划和高优先，无需进入完整编辑
- 规划快照会结合任务优先级、计划档期、剩余专注次数和高效时段生成下一步建议
- 任务页展示任务进度、剩余专注次数、目标达成状态和当前专注状态
- 从任务页开始任务专注后，会自动切回专注页显示倒计时
- 已有专注或待歇息时，会阻止重复开始并引导回专注页；失败后必须先走恢复阶梯才能回到正常专注
- 任务页会展示当前专注/歇息状态，当前任务按钮会切换为「查看」
- 正在专注的任务会锁定编辑、手动完成、归档和删除，避免专注记录与任务进度脱节
- 专注完成生成 `completed` 记录
- 主动结束生成 `abandoned` 记录
- 记录页展示今日 completed 专注时长、完成次数、任务数、计划时长和历史记录
- 记录页支持按全部 / 已完成 / 已结束筛选，并展示完成率、结束次数、平均完成时长
- 记录页按日期分组历史记录，支持删除单条记录和清空全部记录
- 记录页展示本周总专注、活跃日均、最佳日和最近 7 天柱状概览
- 记录页新增复盘洞察，展示最近 30 天完成率、连续专注、高效时段和月度报告商业版入口
- 记录页新增规划复盘，将历史数据和下一轮计划建议连接起来
- 记录页新增本月报告原型，展示月度总时长、完成次数、活跃天数、完成率、平均完成时长、项目投入分布和导出入口
- 记录页按记录阶段分层：无记录只显示第一段引导；30 天内活跃不足 3 天只显示今日/本周洞察和解锁提示；活跃满 3 天解锁月度报告；再完成 3 次带项目归属的专注才展示客户账单
- 任务专注完成或结束后，记录会保留任务所属项目/领域，用于月度项目投入分析
- 专注记录会保留前台专注凭证，离开 App 的次数、时长和守门违规原因会进入历史记录与导出明细；离开或终止的本轮会失败，且不会计入任务进度、今日目标、客户账单或时光币
- 月度报告已抽象为可导出数据结构，包含标题、摘要、洞察、建议、指标矩阵和项目分布
- 记录页新增客户账单 Pro 原型，可按项目自动汇总本月可结算金额、计费时长、项目数量和项目贡献排行
- 客户账单支持按小时费率、最小计费分钟和取整粒度折算金额，只统计已完成专注记录，适合自由职业者/外包/咨询对账场景
- 客户账单支持本月/上月周期切换、项目不可结算标记和发票备注，备注同步到账单卡片和导出 CSV
- CSV 导出核心已准备，可生成月度报告指标、项目投入分布、客户账单和带启动护栏/退出交接/守门模式/白名单完整性/离开审计/恢复阶梯/时光币/小动作次数/重启来源/是否计入专注的专注记录明细四份 CSV 数据
- CSV 导出已接入系统文件选择器（DocumentViewPicker），四份 CSV 可真实保存为文件，带 UTF-8 BOM 防止表格软件乱码
- 月度报告新增最佳连续天数和环比上月投入趋势指标
- 首页下一段建议附带推荐理由（优先级/档期/只差一段）和本段收益提示，与规划核心排序一致
- 任务新建/编辑表单支持常用项目快选 chips，从历史任务和专注记录自动提取
- 设置页支持调整专注/歇息时长、提醒开关、恢复默认节奏和清除本地数据；硬专注守门强制开启，不提供关闭入口
- 设置页支持调整每日目标时长
- 设置页新增客户账单参数，可调整小时费率、最小计费和取整粒度，并支持恢复默认账单设置
- 设置页栖时 Pro 卡片由 ProService 动态渲染能力清单和订阅状态（当前为本地免费版占位）
- 关键失败（提醒/通知/卡片刷新/导出/前台恢复）会写入本地异常日志（上限 50 条），设置页开发调试区可查看和清空
- 已接入 2*2 服务卡片骨架，展示当前状态、剩余时间、任务、今日时长和目标进度
- 通知服务已接入完成链路，系统级延迟提醒使用 `@ohos.reminderAgentManager`，专注到点提醒只提示回到 App 收束确认，不直接宣告后台完成
- 应用图标已同步到 AppScope 和 entry 资源，使用居中的 256px 图标资源

## 本地验证

日常本地验证：

```bash
node scripts/verify.mjs
```

完整验证本地测试、DevEco 构建和设备可见性：

```bash
node scripts/verify.mjs --full
```

也可以单独运行核心逻辑测试：

```bash
node test/timer_core.test.mjs
node test/starter_core.test.mjs
node test/stats_core.test.mjs
node test/planning_core.test.mjs
node test/plan_template_core.test.mjs
node test/monthly_report_core.test.mjs
node test/billing_core.test.mjs
node test/csv_export_core.test.mjs
node test/task_core.test.mjs
```

DevEco/Hvigor 环境可用预检脚本验证：

```bash
node scripts/check-deveco-env.mjs
```

构建并同时检查设备：

```bash
node scripts/check-deveco-env.mjs --assemble --device
```

确认环境通过后可直接构建 HAP：

```bash
node scripts/check-deveco-env.mjs --assemble
```

当前项目要求 `DEVECO_SDK_HOME` 指向 DevEco SDK 根目录，例如 `/Applications/DevEco-Studio.app/Contents/sdk`。如果指到 `sdk/default`，Hvigor 会报 `SDK component missing`。详细排障见 [docs/build-troubleshooting.md](docs/build-troubleshooting.md)。
